Это сопутствующий документ  проекта с описанием методов, классов, ресурсов всего - что сложно комментировать в коде

ремарки по содержимому папки
SQTest\BundleRes собранные в бэндл ресуры
SQTest\TemporalyWorkFiles\BundleResources - исходные ассеты для сборки в бэндл. (Содержит 1 меш и 4 графических файла, на основании которых генерируются текстуры.)
SQTest\Done - собранная версия игры.



Ремарки по запуску.
 - При запуске будет предложено ввести имя и выбрать тип соединения:
		клиент, сервер, Нулевой демо сервер (для запуска на одном компьютере)
 - чтобы скрыть интерфейс во время игры, нажмите "`".
 Игра продолжается до 5 потерянных кружков
 - для запуска из unity3d пользоваться сценой 1 (уровень 0)
  

Ремарки по коду:

Сетевая часть.
NetworkManagerScript Универсальный интерфейс для работы с сетью. Снаружи не имеет значенния клиент или сервер
NetwInterfaceEx - экземпляр класса NetwInterface взаимодейств с с сетью client или server. внутри сериализация/десериализация, чтение запиь в поток, соединение
DemoServerInterfaceEx - экземпляр класса NetwInterface для отладки. Внутри эмуляция 2го компьютера. Выпускает случайные круги в центре своего игрового поля
.. снаружи на них доступа нет только через NetworkManagerScript.

MessageServiceClass.MessageProcessing ("my message");  Работает в т.ч. и из фоновых процессов. отправляет сообщения в GUI.
 
процесс рождения нового кружка: Born New circle
 MainGameScript (instantiate(true)) -> CircleScript -> NetworkManagerScript(serialize) -> NetwInterfaceEx -> [net] -> NetwInterfaceEx -> NetworkManagerScript(deserialize) -> 
 -> NetworkManagerScript.instance.NewDataReceived pull up -> MainGameScript (instantiate(false))
 
Генерация текстур.
За генерацию отвечает ResManagerScript. в его задачу входит генерация по требованию текстур на базе текстур полученных из бэндла и полученного цвета.
Генерация происходит в основном потоке, но в фоновом режиме.
Выгрузка текстур происходит при событии подмены сгенерированных в фоне текстур на рабочие. Одновременно в проекте сохраняется  3 комплекта рабочих текстур, что 
закрывает потребности проекта. Алгоритм Выгрузки не является универсальным и рабочий только в конкретном примере.  Для создания универсального алгоритма, учитывающего индексы
 текстур и привязку данных индексов к объектам ушло бы немного больше времени.
